## 3 · Backend (Render — uses `render.yaml`)

1. Render ▸ **New ▸ Blueprint** ▸ select your repo. It reads `render.yaml` automatically.
2. Fill the vars marked "set in UI":
   - `DATABASE_URL` → the `postgresql+psycopg2://…` string from step 1
   - `CORS_ORIGINS` → your Vercel URL (add after step 4, e.g. `https://crux.vercel.app`)
   - `SUPABASE_SERVICE_ROLE_KEY` → optional (Storage)
   - `GEMINI_API_KEY`, `RESEND_API_KEY` → optional
   - `JWT_SECRET` → auto-generated by Render (or paste the one above)
3. Deploy. When live, open the Render shell once and seed:
   ```bash
   python -m app.seed
   ```
4. Verify: `https://crux-backend.onrender.com/api/health` → `{"status":"healthy"}`

## 4 · Frontend (Vercel)

1. Vercel ▸ **Add New ▸ Project** ▸ import the repo.
2. **Root Directory** → `frontend`. Framework auto-detects Next.js.
3. Environment variable:
   ```
   NEXT_PUBLIC_API_URL = https://crux-backend.onrender.com
   ```
4. Deploy. Then copy the Vercel URL back into Render's `CORS_ORIGINS` and redeploy the backend.
